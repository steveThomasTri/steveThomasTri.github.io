Part 3: Data Acquisition

How do we gather the data needed to decide on whether someone will be drafted?

You can find all of the jupyter notebooks and the csv files that I have used on GitHub

In order to acquire the data that we would like to model, we needed to go to two websites.

  • nflcombineresults.com
  • drafthistory.com

The reason that we chose the first website is considering all other websites at the time, this one seems to be the most comprehensive. Whilst pro-football-reference had combine results that went back to 2000, The nflcombineresults.com went all the way back to 1997. From the first jupyter notebook (1 Data Acquisition) we start the process of getting all of the combine results from 1997 – 2018. The reason that we went back to 1997 is because that year first introduced the 3 cone event. The website is nflcombineresults.com – 32 years of NFL scouting Combine data is the website that we used to scrape the data from splinter. After I got all of the combine data, I had to normalize the names column so that the name is lower case and remove all punctuations marks so that when we compare the names to whether they get drafted or not, the process would be easier. For this, we go to drafthistory.com. Gather the list of names from 1997 - 2018 and then cross check them to the nflcombinedata97to18.csv file. We would also like to remove rows that didn’t post a score in any of the events. Our model assumes that a prospect has competed in at least one event. So in this case, we will now revise our main question.

Having at least participated in one NFL combine event, will a future prospect be drafted in the NFL?

*A moment of retrospect* In gathering the data, you will notice that I used splinter to gather all of the data for each year. If I had to do this process all over again, It is much easier to use the pandas read_html function as it is much quicker to get the data and then each year I can concatenate the dataframes. Also, there may have been two key columns that were missing, hand size and arm length. These factors could have affected any future models that I might have made.

Also, in 2015 the data accounted for 700 prospects in the NFL combine for that year. This is caused to additional data being added for that year. We cannot tell the reason as to why this occurred, but we cross-checked the number of combine participants for 2015 and most of the sources have the number at around 300 individuals. So well will need to remove those rows that are not in the combine. We will be going to the SB Nation website that has the list of 2015 combine participants and search the dataframe to see if any players do not make the list. If the name is not in the article, remove that row and only search the rows with year 2015. In doing so, we have eliminated 451 rows giving a final 2015 total of 285 rows. Then we merged the two dataframes by name and year to obtain a new dataset of 6300 rows.

The drafthistory.com website provides us with the years of every NFL player that has been drafted. All that we are looking for is if the player is on the list, then we consider them drafted. Going to the website, I used splinter to gather a list of each player. And those players are stored in a dataframe called drafted. Storing the name, year, and a 1 for drafted. Once I have this dataframe, I will merge this dataframe with the combine information that I have previously made and merge it on the name and the year, because it is possible that there may be players with the same name in different years. For the column drafted, I changed the values of 1 to Y and 0 to N to get the final csv file with 6,300 rows and 13 columns. The completed file is called cleaned_NFL_combine_97_to_2018.csv. We now have the necessary data that has been cleaned, we will now go into analyzing our data and see what we can learn and any trends that we can observe.
