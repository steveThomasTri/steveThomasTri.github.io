Part 7: Feature Selection/Feature Engineering

Feature Engineering is taking the data that you have already to generate new features. Sometimes it is an attempt to explain features not considered in the model. For example, the feature that we want to add is BMI (Body Mass Index) we can use the prospect's height and weight to generate what their BMI is so that we may be able to predict if someone is drafted. You can see that you can generate any number of additional features based on the data you have, but the risk is that you are going to introduce unnecessary noise to your model. Feature engineering is an art and a science, so the only way to know what constitutes a good feature is by experience.

What we will do is to categorize the data that we have into ten groups with ten separate csvs. These ten categories are:

  • Offensive Line (OL) consists of OT, OG, C, OL
  • Defensive Line (DL) consists of DT, DE, DL
  • Running Backs (RB) consists of RB, HB, FB
  • Line Backers (LB) consists of OLB, ILB, LB, DB
  • Tight Ends (TE)
  • Cornerbacks (CB)
  • Wide Receivers (WR)
  • Safety (S) consists of FS, SS
  • Quarterbacks (QB)
  • Punters (P)

So let's revise our question:

Based on an NFL draft prospects height, weight, and the results that they posted in the NFL combine, will this person be drafted if we group them by position?

So we get away from a position neutral model to a model based on a players declared position in the combine. We start off separating each player in the training data by the categories. We will be observing the data of each group. 6 Splitting into categories is the notebook that makes those ten csvs in the TrainTest Folder. Within the TrainTest folder, we make ten folders that coorespond to the groups.

For each folder, there will be three notebooks and a csv file of training data for that group.

The next part, models, breaks down the ten charts. Starting from the left, Using the original 8 features, we get the base accuracy and a possible accuracy if any of the 8 features are removed. The next line is any features that we will add. For different groups, there will be certain featurs that make sense to add like wide receivers receiving yards, touchdowns, that do not for a defense tackle. After we add more features, we run the model again and this time we get the highest accuracy attainable if we do not do any feature tuning and any features that we will remove.

The second table is once we get the maximum accuracy, we will be doing hyperparameter tuning to see if we can get the model to perform better. There are 6 parameters that we have considered tuning:

  • Learning Rate - In each step, as new data is taken in, the model will change how certain features are weighed in each step. The larger the value, the more rows it learns quickly.
  • n_estimators - Number of decision trees
  • Max Depth - How deep is each tree
  • Minimun Child Weight - A larger value makes the model more conservative
  • Gamma - Minimum loss reduction required to make further partitions on a leaf node of the tree.
  • Colsample_bytree - subsample ratio of columns when constructing each tree.

For more information on each of these parameters, consult the documentation. XGBoost Parameters

Here is a breakdown of what each notebook was used for:

  • The first notebook establishes the model from the base configurations and finds the maximum accuracy if the original 8 features are considered.
  • Part II Notebook uses splinter to obtain additional data based on the group. It goes through each name and fills in the data and if they got awards. College stats are left blank if none exists, but awards are always populated.
  • Part III notebook gets the base accuracy when we add features from part II. Then we find the maximum accuracy if certain features are removed. Verify that removing those features give us the maximum accuracy. Then we implement hyperparameter tuning using the 6 parameters through brute force. There are 14,400 different combinations to check for.
